# منصة "مكانك" للتجارة الإلكترونية | Your Place E-Commerce Platform

منصة تجارة إلكترونية عربية/إنجليزية متكاملة وسريعة (لوحة تحكم + واجهة متجر + نظام طلب سريع + ربط واتساب + حماية كاملة) مصممة لحملات الشراء السريع والدفع عند الاستلام (COD) مشابهة لمنصة Easy Order ومصممة بأحدث التقنيات وأقوى معايير الأمان.

---

## ✨ المميزات الرئيسية (Core Features)

* **لوحة تحكم متكاملة:** إدارة المنتجات، التصنيفات، الطلبات، العملاء، والكوبونات.
* **صفحة هبوط مخصصة لكل منتج:** تصميم جذاب يزيد من نسبة الشراء (Conversion Rate) مع عداد تنازلي وبطاقات آراء العملاء.
* **نموذج طلب سريع (4 خانات):** تبسيط عملية الشراء للعميل (الاسم، الموبايل، العنوان، الكمية).
* **احتساب ديناميكي وتحديث يدوي للشحن:** المجموع الفرعي = الكمية × السعر، ويضاف مصاريف الشحن يدوياً لكل طلب من لوحة التحكم لتحديث الإجمالي النهائي ديناميكياً (لا توجد أي قيم ثابتة للشحن بالكود).
* **إتمام الطلب عبر واتساب:** إرسال نص عربي/إنجليزي منسق بالكامل يحتوي على بيانات الطلب إلى رقم واتساب المتجر تلقائياً، بالإضافة إلى خيار الدفع عند الاستلام.
* **تعدد اللغات والاتجاهات (RTL/LTR):** دعم كامل للغة العربية (dir="rtl") والإنجليزية (dir="ltr") مع خطوط Cairo و Inter مدمجة.
* **إدارة المخزون والتنبيهات:** تنبيه فوري في لوحة التحكم عند وصول مخزون أي منتج للحد الأدنى.
* **سجل الرقابة والأمان (Audit Logs):** تتبع تفصيلي لأي عمليات مشبوهة أو دخول خاطئ مع تسجيل الـ IP والمتصفح.

## 🔒 طبقات الحماية وتأمين النظام (Security Shield)

1. **منع حقن الاستعلامات (SQL Injection):** استخدام الاستعلامات المجهزة (Prepared Statements) في قاعدة بيانات SQLite بالكامل.
2. **حماية XSS (Cross-Site Scripting):** فلترة وتعقيم المدخلات في Express وتقديم ترميز آمن في React، مع تفعيل سياسة أمان المحتوى عبر خوذة الأمان `helmet`.
3. **منع هجمات CSRF:** مصادقة الجلسات عبر ترويسات `Authorization: Bearer <JWT>` المحمية.
4. **تأمين كلمات المرور:** استخدام تشفير وتمليح `bcryptjs` بـ 12 دورة تشفير لحسابات الموظفين.
5. **أمان رفع الملفات:** التحقق من صيغ وحجم صور المنتجات المرفوعة وإعادة تسميتها عشوائياً لمنع حقن السكريبتات التنفيذية في مجلد `uploads/`.
6. **التحكم بالصلاحيات (RBAC):** تحديد أدوار الوصول (مسؤول Admin، مدير عمليات Manager، موظف Staff).

## 🚀 تشغيل المشروع محلياً (Local Development)

### المتطلبات الأساسية
* تأكد من تثبيت **Node.js** (إصدار 18 فما فوق).

### الخطوة 1: تشغيل خادم الخلفية (Backend)
1. افتح مبوب أوامر جديد في مجلد `backend`:
   ```bash
   cd backend
   ```
2. قم بتثبيت المكتبات المطلوبة (تم ذلك بالفعل):
   ```bash
   npm install
   ```
3. قم بتهيئة قاعدة البيانات وحقن البيانات الافتراضية (تم ذلك بالفعل):
   ```bash
   npm run seed
   ```
4. ابدأ تشغيل خادم الويب:
   ```bash
   npm start
   ```
   * سيعمل الخادم على المنفذ: `http://localhost:5000`

### الخطوة 2: تشغيل الواجهة الأمامية (Frontend)
1. افتح مبوب أوامر جديد في مجلد `frontend`:
   ```bash
   cd frontend
   ```
2. قم بتثبيت المكتبات المطلوبة (تم ذلك بالفعل):
   ```bash
   npm install
   ```
3. ابدأ تشغيل خادم التطوير الخاص بـ Vite:
   ```bash
   npm run dev
   ```
   * سيعمل التطبيق على الرابط: `http://localhost:3000`

## 🌐 دليل النشر والاستضافة في الإنتاج (Production Deployment Guide)

لنشر المنصة للجمهور على خادم سحابي أو خادم خاص (VPS)، اتبع ما يلي:

### الخيار 1: النشر على خادم خاص (VPS Linux - Ubuntu)
1. **تجهيز السيرفر:** قم بتثبيت Node.js و SQLite3 و Nginx.
2. **رفع الكود:** انسخ مجلد المشروع إلى السيرفر عبر Git أو SFTP.
3. **بناء الواجهة الأمامية:**
   * في مجلد `frontend` قم بتشغيل: `npm run build`.
   * سيقوم Vite بتوليد مجلد `dist/` الذي يحتوي على ملفات HTML/JS/CSS المبنية والمضغوطة.
4. **تجهيز الخلفية لخدمة الواجهة:**
   * قم بنقل ملفات مجلد `dist/` إلى خادم الويب أو قم بتهيئة Nginx لخدمة مجلد `dist` مباشرة، وتوجيه طلبات `/api` إلى خادم Node.js العامل بالخلفية عبر `proxy_pass`.
5. **إدارة تشغيل الخادم:**
   * استخدم أداة **PM2** لإبقاء خادم Node.js يعمل في الخلفية باستمرار حتى بعد إغلاق التيرمينال:
     ```bash
     npm install -g pm2
     pm2 start server.js --name "your-place-backend"
     pm2 save
     pm2 startup
     ```
6. **إعداد جدار الحماية وعقد شهادة SSL:**
   * قم بتفعيل Let's Encrypt لحماية النطاق بـ HTTPS مجاناً لتأمين اتصالات العملاء والمديرين بالكامل.

### الخيار 2: النشر على منصات سحابية بدون خوادم (PaaS - Render / Railway)
1. **قاعدة البيانات:** نظراً لأن SQLite عبارة عن ملف محلي (`database.sqlite`)، يجب استخدام **Persistent Volume** (مساحة تخزين دائمة) على Render أو Railway لضمان عدم حذف قاعدة البيانات وصور المنتجات المرفوعة عند إعادة تشغيل الحاوية.
2. **الرفع:** اربط مستودع GitHub الخاص بك بالمنصة السحابية وقم بتهيئة أوامر التشغيل والبناء تلقائياً.
